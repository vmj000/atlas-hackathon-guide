MongoDB Compass

Visual GUI — explore, query & aggregate without writing a line of code.

  • Download & InstallGet Compass from mongodb.com/products/compass. Available for macOS, Windows and Linux. Takes ~2 min.
  • Paste your connection stringOpen Compass → New Connection → paste your mongodb+srv://... URI. Optionally name it and save it for next time.
  • Explore your dataBrowse collections, view documents, inspect schema distribution, and spot index opportunities immediately.
  • Use the Aggregation BuilderClick the Aggregations tab → Add Stage → pick an operator ($match, $group, $lookup …). Preview output at every stage in real time.
  • Export pipeline as codeClick Export to Language in the builder to get Node.js / Python / Java / C# code instantly — no hand-writing aggregations.
💡

Hackathon tip: Design your aggregation pipeline visually in Compass, then export it as driver code — this alone can save 30–60 minutes of trial-and-error.

mongosh — MongoDB Shell

The modern JS shell. Seed data, run admin ops, and script everything from your terminal.

  • Install mongoshmacOS: brew install mongosh · npm: npm i -g mongosh · or download from mongodb.com/try/download/shell
  • Connect to AtlasRun the connect command below with your SRV connection string. Optionally embed credentials or enter password when prompted.
  • Switch database & queryType use mydb to switch context, then use full JavaScript — loops, variables, helper functions — alongside MongoDB commands.
  • Run a seed scriptPass a .js file directly: mongosh "mongodb+srv://..." seed.js — perfect for pre-loading demo data.
Terminal
# Connect mongosh "mongodb+srv://user:pass@cluster0.abc.mongodb.net/hackdb" # Switch db and CRUD use hackdb db.users.insertOne({ name: "Alice", score: 42 }) db.users.find({ score: { $gte: 30 } }) db.users.updateOne({ name: "Alice" }, { $set: { score: 99 } }) db.users.deleteOne({ name: "Alice" }) # Seed data from a JS file mongosh "mongodb+srv://..." seed.js
seed.js — Seed Script Example
use("hackdb"); db.products.insertMany([ { name: "Widget A", price: 9.99, stock: 100 }, { name: "Widget B", price: 14.99, stock: 50 }, ]); print("✅ Seeded!");

Node.js — Quick Start
import { MongoClient } from "mongodb"; import * as dotenv from "dotenv"; dotenv.config(); const client = new MongoClient(process.env.MONGODB_URI); async function main() { await client.connect(); const db = client.db("hackdb"); const col = db.collection("users"); // Insert await col.insertOne({ name: "Alice", ts: new Date() }); // Query const docs = await col.find({ name: "Alice" }).toArray(); console.log(docs); // Aggregation const results = await col.aggregate([ { $group: { _id: "$region", count: { $sum: 1 } } } ]).toArray(); } main().finally(() => client.close());
Python (PyMongo) — Quick Start
from pymongo import MongoClient from dotenv import load_dotenv import os load_dotenv() client = MongoClient(os.environ["MONGODB_URI"]) db = client["hackdb"] col = db["users"] # Insert col.insert_one({"name": "Alice", "score": 42}) # Query for doc in col.find({"score": {"$gte": 30}}): print(doc) # Update col.update_one({"name": "Alice"}, {"$set": {"score": 99}})

MCP & AI Agents

Connect Claude, Cursor, VS Code Copilot directly to your Atlas cluster via the MongoDB MCP server.

MCP (Model Context Protocol) is an open standard that lets AI coding assistants talk directly to external tools and databases — including MongoDB Atlas. Set it up once and your AI assistant can explore schemas, generate queries, seed data, and suggest indexes using plain English.

Step 1 — Get Atlas API Keys

  • Create an API KeyAtlas UI → Access Manager → API Keys → Create API Key. Assign Project Read Write role. Save the public + private keys.
  • Whitelist your IPUnder the key's Access List, add your current IP (or 0.0.0.0/0 for hackathon convenience). Never use 0.0.0.0/0 in production.
  • Store in .envAdd MDB_ATLAS_PUBLIC_KEY, MDB_ATLAS_PRIVATE_KEY, and MDB_CONNECTION_STRING to your .env file.

Step 2 — Install the MongoDB MCP Server

Terminal
# Use via npx (no global install needed — recommended for hackathons) npx -y @mongodb-js/mongodb-mcp-server # Or install globally npm install -g @mongodb-js/mongodb-mcp-server

Step 3 — Configure in Claude Desktop

~/.config/claude/claude_desktop_config.json (macOS: ~/Library/Application Support/Claude/)
{ "mcpServers": { "mongodb": { "command": "npx", "args": ["-y", "@mongodb-js/mongodb-mcp-server"], "env": { "MDB_CONNECTION_STRING": "mongodb+srv://user:pass@cluster.mongodb.net/hackdb", "MDB_ATLAS_PUBLIC_KEY": "<your-public-key>", "MDB_ATLAS_PRIVATE_KEY": "<your-private-key>" } } } }

Step 4 — Configure in Cursor / VS Code

.cursor/mcp.json OR .vscode/mcp.json (project-level)
{ "servers": { "mongodb": { "type": "stdio", "command": "npx", "args": ["-y", "@mongodb-js/mongodb-mcp-server"], "env": { "MDB_CONNECTION_STRING": "${env:MONGODB_URI}" } } } }

Quick Tips & Cheatsheet

The stuff that saves hours in a hackathon.

🆓 Free M0 Tier

Use Atlas M0 (free forever, no credit card). 512 MB storage, shared cluster. More than enough for hackathon workloads.

🌐 Allow All IPs

During hacking, set Network Access to 0.0.0.0/0 to avoid IP whitelist friction. Lock it down before going live.

🔎 Atlas Search

Enable full-text search in one click from the Atlas UI. No Elasticsearch needed — use $search in aggregation.

📡 Change Streams

Watch for real-time data changes with collection.watch(). Live dashboards and event-driven triggers made easy.

🗺 Atlas Charts

Build embeddable dashboards on top of your data without writing frontend code — straight from the Atlas UI.

⚡ Vector Search

Atlas Vector Search lets you store embeddings and run semantic similarity search — great for AI features in your hackathon app.

⚠️

Security reminder: Store your MONGODB_URI and Atlas API keys in a .env file and add it to .gitignore. Credentials exposed in a public repo will be exploited within minutes by automated scanners.

Sample datasets: Atlas has free built-in sample data (movies, AirBnB listings, stocks, etc.). In the Atlas UI, click your cluster → Load Sample Dataset. Instant realistic data for your demos.
